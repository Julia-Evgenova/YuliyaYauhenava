HTML <picture> Тег


HTML-элемент <picture> служит контейнером для одного или более элементов <source> и одного элемента <img> для обеспечения оптимальной версии изображения для различных размеров экрана. Браузер рассмотрит каждый из дочерних элементов <source> и выберет один, соответствующий лучшему совпадению; если совпадений среди элементов <source> найдено не будет, то будет выбран файл, указанный атрибутом src элемента <img> . Затем выбранное изображение отображается в пространстве, занятом элементом <img>.

Чтобы выбрать оптимальное изображение, user agent анализирует атрибуты srcset, media, и type элемента <source> и выбирает совместимое изображение, которое наилучшим образом соответствует текущему макету страницы, характеристикам устройства отображения и т. д.

Элемент <picture> часто используют для того, чтобы обеспечить HiDPI (Retina)-версию изображений для дисплеев с высоким разрешением, а также для добавления других форматов изображений, не поддерживаемых всеми браузерами.

Тег <picture> используется для отображения изображений с альтернативными источниками в зависимости от разрешения экрана. Это позволяет браузеру выбирать наиболее подходящее изображение для текущего экрана и загружать только его, что уменьшает время загрузки страницы и улучшает производительность.


Отличия от тега <img>:


Ter <picture> отличается от <img> тем, что позволяет предоставлять несколько вариантов изображений для выбора браузером в зависимости от определенных условий, таких как размер экрана, разрешение и т. д. В то время как тег <img> просто встраивает одно изображение на веб-страницу.

Основные различия:

  • Гибкость в выборе изображений: Тег <picture> позволяет вам предоставить разные источники изображений для различных сценариев, например, для разных размеров экрана или разрешений. Это позволяет создавать более адаптивные и эффективные веб-страницы.
  • Поддержка медиа-запросов: Вы можете использовать медиа-запросы внутри элементов <source> для определения, какие изображения следует использовать в зависимости от характеристик устройства, таких как ширина экрана, разрешение, ориентация и т. д.
  • Функциональность альтернативных изображений: Даже если браузер не поддерживает тег <picture>, он всегда может отображать обычный тег <img> внутри элемента <picture>. Это обеспечивает обратную совместимость с более старыми браузерами. Таким образом, тег <picture> предоставляет более гибкий и адаптивный подход к встраиванию изображений на веб-страницу по сравнению с простым тегом <img>.

Вот несколько дополнительных нюансов, которые стоит учитывать при использовании тera <picture>:

  • Поддержка браузерами: Некоторые старые версии браузеров могут не поддерживать тег <picture> полностью или не поддерживать некоторые его атрибуты, такие как media в элементе <source>. Поэтому важно проверить совместимость с целевыми браузерами и предусмотреть резервные варианты для неподдерживаемых случаев.
  • Атрибуты sizes и type: Вы можете использовать атрибут sizes для определения размеров изображения на различных экранах, а атрибут type для указания типа МІМЕ изображения. Эти атрибуты могут быть полезны для дополнительной настройки отображения изображений.
  • SEO-оптимизация: обратите внимание, что контент, вложенный внутри тега <picture>, не будет индексироваться поисковыми системами, если он не содержит альтернативный текст в теге <img>. Убедитесь, что ваши изображения имеют адекватные и описательные атрибуты alt.
  • Прогрессивное улучшение (Progressive Enhancement): Важно использовать тег <picture> с учётом принципа прогрессивного улучшения, чтобы основное содержимое (обычно <img>) всегда было доступно для всех пользователей, в том числе для тех, у кого отключены JavaScript или не поддерживается функциональность тега <picture>.

Учитывая эти нюансы, использование тега <picture> может значительно улучшить адаптивность и производительность вашего веб-сайта, обеспечивая лучший пользовательский опыт на различных устройствах и разрешениях экранов.


Примеры использования:


Отображение разных изображений в зависимости от ширины экрана:

            
            <picture>

            <source srcset="image-medium.jpg" media="(max-width: 1024px)">

            <source srcset="image-large.jpg">

            <img src="image-default.jpg" alt="Image description">

            </picture>
        
    
        
            <picture>
            
            <source media="(min-width: 1200px)" srcset="large-image.jpg">
            
            <source media="(min-width: 800px)" srcset=medium-image.jpg"">
            
            <img src="small-image.jpg" alt="Описание изображения">
    
            </picture>
  • Если ширина экрана больше 1200рх, будет загружено изображение "large-image. jpg".
  • Если ширина экрана между 800рх и 1200рх, будет загружено изображение "medium-image.jpg".
  • Если ни одно из условий не выполняется или браузер не поддерживает тег <picture> будет загружено изображение 'small-image.jpg.
  • <source> определяет различные варианты источников изображений в зависимости от медиа-запросов, указанных в атрибуте media.
  • srcset указывает пути к изображениям для каждого источника.
  • <img> является резервным вариантом и будет использован, если ни один из элементов
  • <source> не соответствует текущему устройству или если браузер не поддерживает тег <picture>.

Отображение разных форматов изображений в зависимости от браузера:

            
            <picture>

            <source srcset ="image.webp" type="image/webp">

            <source srcset="image.jpg" type="image/jpeg">

            <img src="image.png" alt="Image description">

            </picture>

Отображение фонового изображения с альтернативным источником:

            
            <div> style="background-image: url('image-default.jpg');">

            <picture>

            <source srcset="image-large.jpg">

            <img src="image-default.jpg" alt="Image description">

            </picture>

            </div>
